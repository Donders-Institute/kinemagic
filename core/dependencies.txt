DEPENDENCIES ON MATLAB TOOLBOXES

CRITICAL
km_filter.m depends on
  signal: butter, cheby1, cheby2, fir1, filtfilt: these functions are essential

DEBATABLE
cutoutlier.m depends on
  stats: prctile: only when you use the setting 'iqr' (I actually prefer to cut outliers based on the inter-quartile-range (iqr) instead of the standard deviation)

NOT CRITICAL (can be circumvented)
km_movdef.m depends on
  image: medfilt2: only when you use the setting 'compwin' (this can be averted, but is not yet implemented)
  stats: betapdf: only when you use the setting 'probwin' (this is very unlikely and very advanced)
  stats: nanmedian: a replacement function for nanmedian can be easily created
km_plot_descriptives.m depends on
  stats: nanmedian: a replacement function for nanmedian can be easily created
km_trlrej.m depends on
  stats: nanmedian: a replacement function for nanmedian can be easily created
km_getaxrange.m depends on
  stats: nanmedian: a replacement function for nanmedian can be easily created

REDUNDANT (is already circumvented when toolbox is not present)
export_fig.m depends on
  image: imresize, imwrite: these functions are faster than the default ones, but are not essential
km_mdCI.m depends on
  stats: finv: only when you use the f-distribution (the f-dist is an extra option, chi-squared is the default)
